# Web UI 고도화 계획 (3차) — 뷰어, 그래프, 연결성

> 상태: **구현 완료** (2026-07-24, M9–M10+). 설계 결정 기록으로 보존.
> 선행: [web-ui-plan.md](web-ui-plan.md)(M1–M4) · [web-ui-plan-2.md](web-ui-plan-2.md)(M5–M8).
> 실행 기록: [web-ui-board.md](web-ui-board.md).
>
> 1·2차가 "위키를 읽고 쓰고, 위키가 말을 거는" 골격이었다면, 3차는 두 방향의
> 완성이다: **읽는 경험을 현대적 뷰어 수준으로** 끌어올리고, **연결(링크 그래프)
> 자체를 일급 대상**으로 승격시킨다.

## 1. 문제 정의

1. **뷰어가 빈약했다.** 코드 블록에 문법 하이라이팅이 없고, mermaid 다이어그램과
   LaTeX 수식이 원문 그대로 노출됐다. 기술 위키의 본문이 제대로 읽히지 않으면
   "웹 UI가 1차 접점"이라는 전제가 무너진다.
2. **연결을 전역으로 볼 수 없었다.** 페이지별 백링크·로컬 그래프는 있었지만
   Obsidian Graph View 같은 전체 조망이 없어, 지식 그물의 형태(허브, 변두리,
   끊어진 곳)를 인지할 수 없었다.
3. **전역 조망이 생기자마자 새 문제가 보였다: 섬.** 서로는 링크됐지만 본토와
   끊어진 클러스터는 고아 검사를 전부 통과한다 — 기존 lint의 사각지대.
   (도구가 새 문제를 드러내고 그 도구로 해법을 검증한, 이 프로젝트다운 순서였다.)

## 2. 설계 결정

### D1. 문법 하이라이팅은 서버사이드 (chroma)

- highlight.js류 클라이언트 하이라이팅 대신 goldmark-highlighting + chroma로
  **렌더 시점에 토큰화**한다. 첫 페인트부터 색이 입혀져 있고, 클라이언트 JS
  비용이 0이며, JS 없는 환경에서도 동작한다.
- 색은 인라인이 아니라 CSS 클래스로 (`chroma.WithClasses`) — `static/chroma.css`
  하나로 라이트(github)/다크(github-dark)를 `prefers-color-scheme`에 따라 전환.

### D2. 클라이언트 라이브러리는 vendor — CDN 금지

- mermaid(3.4MB)·MathJax tex-svg(2MB)·force-graph(173KB)는 클라이언트 렌더가
  불가피하다(SVG 생성·물리 시뮬레이션). 이들을 CDN이 아니라 **embed.FS로
  바이너리에 내장**한다 — philosophy.md 비-목표 "외부 서비스 의존 없음"의 웹
  확장이다. 오프라인·사설망(tailscale)에서 완결되고, 버전이 바이너리에 고정된다.
- 비용(바이너리 +5.5MB)은 lazy 로딩으로 상쇄: mermaid는 다이어그램이 있는
  페이지에서만, MathJax는 본문(코드 제외)에 TeX 패턴이 보일 때만 로드된다.
- MathJax는 tex-chtml이 아니라 **tex-svg**를 쓴다 — 폰트가 SVG 경로로 내장돼
  단일 파일로 완결되기 때문(chtml은 웹폰트 파일 수십 개를 따로 요구한다).

### D3. 뷰어의 나머지는 표준 관례를 따른다

코드 복사 버튼(hover), 헤딩 앵커(#), 외부 링크 새 탭+↗, footnote — 현대
마크다운 뷰어의 관례이므로 발명 없이 채택. 구현은 전부 vanilla JS 수십 줄.

### D4. 전역 그래프는 캔버스 force 레이아웃 (force-graph)

- 149노드/564엣지(그리고 수천까지)를 60fps로 줌/팬/드래그하려면 SVG DOM이
  아니라 캔버스가 맞다. d3-force를 직접 조립하는 대신 force-graph(단일 번들,
  줌/팬/드래그/호버 내장)를 vendor로 채택 — D2 정책 적용.
- 그래프는 단순 시각화가 아니라 **상태의 지도**다: 색=디렉토리, 크기=연결 수,
  붉은 링=안 읽음(원칙 12의 읽기 이력), 노란 점선 halo=섬(D6). 위키의 건강이
  한 화면에서 보이게 한다.
- 라벨은 줌 레벨에 반응(Obsidian 동작)하되, **hover 노드는 줌과 무관하게 정보
  카드**(제목·분류·링크 수·읽음·발췌)를 띄운다 — 줌아웃하면 점들의 모임이라
  정보량이 0이 되는 문제의 해법.

### D5. 툴팁 데이터는 선적재한다

- 처음엔 hover 시 `/api/preview`를 비동기 fetch했으나, hover 깜빡임과 비동기
  갱신이 경합해 발췌문이 누락되는 사례가 실브라우저 검증에서 발견됐다.
- 원인 추적 대신 구조를 바꿨다: **발췌문을 `/api/graph` 페이로드에 포함**
  (149페이지에 ~30KB, 서버는 이미 본문을 파싱해 둔 상태). fetch/캐시/갱신
  코드가 통째로 사라지고 툴팁이 항상 원자적으로 완성된다.
  경합하는 비동기를 고치는 최선의 방법은 비동기를 없애는 것이었다.

### D6. 섬: 정의는 결정론, 수리는 판단

- **정의**: 무향 위키링크 그래프의 연결 성분 중 최대 성분(본토) 밖의 모든
  성분. "index에서 출발"은 기준이 될 수 없다 — 생성되는 index/*.md는 모든
  페이지를 기계적으로 나열하므로 형식적 도달성은 항상 참이고, 의미 있는 것은
  유기적 링크의 도달 가능성이다. 최대 성분 기준은 설정이 필요 없고 결정론적이다.
- **검출**은 lint(`island` finding, invariants B5)로 — "문서에만 있고 점검 안
  되는 규칙"이 되지 않게. **수리**는 원칙 10(확언/추측 구분)을 따라 다리
  후보(섬↔본토 최고 코사인 페어) 제시까지만 — 요리 레시피 클러스터처럼
  "의도된 섬"이 실재하므로 자동 연결은 하지 않는다.

  다리 제안의 타당성 확인.
